CAMERAContext

Definition

#include <twl/camera.h>


typedef enum
{
    CAMERA_CONTEXT_A     = (1<<0),
    CAMERA_CONTEXT_B     = (1<<1),
    CAMERA_CONTEXT_BOTH  = (CAMERA_CONTEXT_A|CAMERA_CONTEXT_B)
}
CAMERAContext;

Elements

CAMERA_CONTEXT_A Camera setting context A.
CAMERA_CONTEXT_B Camera setting context B.
CAMERA_CONTEXT_BOTH Both context A and B.

Description

An enumerated type that specifies the camera setting context.

Think of a "context" here as a place to save the state for camera settings.
The library currently allows you to save combinations of settings for the resolution, effect, and flipping in two contexts: A and B. By default, both the inner and outer camera use context A when started. See the section on default values in the Overview for more details.

Contexts are used as follows. If the CAMERA_I2CSizeEx, CAMERA_I2CEffectEx, and CAMERA_I2CFlipEx functions are used to set the necessary values in context B, the CAMERA_I2CContextSwitch function is then called to switch the current context to context B when those settings are required. It is faster to switch settings in this way than to individually change camera settings on demand.
