Changes in 2.1.0

Note 81-K01: Fixed NITRO-Player startup bug

When NITRO-Player was already running and the Start NITRO-Player button was pressed while using a Sound project stored in a Japanese-language folder, the message XXXX contains an invalid path would appear on NITRO-Player, and an error would occur. That bug has been fixed.

Changes in Version 2008/04/08

Note 76-K01: (Manual) Changed the format of the Revision History and revised the page headers

Changed the format of the Revision History in the document NITRO-SoundMaker_UserManual.pdf and revised the page headers.

Changes in Version 2007/11/26

Note 71-K01: Added support for disabling envelope release

Added support for disabling envelope release phase. Disable the envelope release process when the note is off for regions and instruments specified as disabled and continue playback until the end of the wave file.

Note 71-K02: Added support for MIDI RPN

Added support for MIDI RPN. With RPN LSB = 0, MSB = 0, it is possible to set pitch bend sensitivity (bendrange).

Note 71-K03: Improved ADPCM encoding process

Made improvement to mitigate noise generated by wave form file ADPCM encoding.

Note 71-K04: Corrected problem related to the AIFF loading process

Fixed a conversion error problem despite its being the correct AIFF file.

Note 71-K05: Fixed a problem related to displaying the file path for the reference waveform of the bank window.

Fixed a problem where the path of the wave form file that is referenced by the region selected in the sample map would not be displayed at the bottom of the window.

Note 71-K06: Fixed a problem related to All Region checkbox

Fixed a problem where the All Region checkbox on the instrument parameter panel would remain selected when specifying a different envelope in multiple regions from the percussion list.

Note 71-K07: Fixed a bug related to the File Selection dialog box for sequences

When the File Selection dialog box opened from the Create New Sequence dialog box, the file type would be restricted to .smft. This was revised to change according to the file's type (for example, to .mid).

Note 71-K08: Fixed a problem in the operation to recover (CTRL+Z) the region split

Fixed a problem where application errors are generated when repeating the operation (CTRL+Z) to return the split region to its original state in the sample map.

Changes in Version 2007/03/14

Note 64-K01: Added user command execution feature before and after conversion.

The commands run before and after the conversion can now be registered with the Project Settings.

Note 64-K02: Added converting feature by specifying the file type

It is now possible to shorten the conversion time by specifying the file type when converting. It is possible to specify using the Convert by Specifying the Option menu.

Note 64-K03: Revised WGroup setting of the multi-region instrument

Conventionally, it has been possible to specify WGroup for only a single region, but it is now possible to also specify for multi regions.

Note 64-K04: Revised original key when region is split in the percussion list

Previously the new regions created by region split in the percussion list inherited the original keys of the source region. This has been corrected so that the new regions will have the same keys as where the split occurred.

Note 64-K05: Revised SeqMax when creating a new player

Conventionally the value of SeqMax when a new player was created was set to 0, but this was corrected to be set to 1.

Note 64-K06: Fixed problem relating to Undo of deleting the bank

Corrected a problem where an application error was generated when saving a project after undoing a deletion of a bank.

Note 64-K07: Fixed problem relating to write-prohibit files

Fixed a problem where application errors were generated when executing a conversion with a write-prohibit file in an edited state.

Note 64-K08: Fixed problem relating to bank files whose release has been set to DISABLE

Fixed a problem where application errors were generated when loading a bank file whose release was set to DISABLE. Note that even in this version, it is not possible to set release to DISABLE from the NITRO-SoundMaker. To set release to DISABLE, close the bank window opened in NITRO-SoundMaker and use a text editor to edit the bank file. For details on editing the bank file, see the NITRO-Composer Bank Data Manual.

Changes in Version 2006/03/15

Note 56-K01: Fixed problem where invalid data reference information is output

Fixed a problem that invalid data reference information was output when using multiple sound sets or when using a fix-index-number feature.

Note 56-K02: Fixed problem where invalid data references were generated when creating a new bank

Fixed a problem where reference information to an invalid waveform archive was generated when a new reference waveform archive was created at the same time when a new bank was generated while using multiple sound sets.

Note 56-K03: Fixed problem relating to the volume of PC emulation playback

During PC emulation playback, the volume of the right channel was output at higher-than-normal volume. We fixed this bug.

Note 56-K04: Fixed problem where the convert log would not be updated

Fixed a problem where the log would not be updated when the number of convert log characters reaches a certain number.

Changes in Version 2006/01/18

Note 55-K01: Fixed problem where invalid group reference information would be output when using multiple sound sets

Fixed problem where invalid group reference information was output when using multiple sound sets.

Note 55-K02: Fixed problem where it was not possible to convert when referencing data between multiple sound sets

Fixed a problem where conversion errors were generated and conversion was not possible when referencing data from one sound set to another sound set in a project using multiple sound sets.

Note 55-K03: Fixed problem in saving the instrument region

Fixed a problem where the last region was not correctly saved when more than nine instrument regions were created.

Note 55-K04: Corrected a problem with reading AIFF markers

Fixed a problem where reading files would fail and sounds could not be played back when AIFF markers included characters other than ASCII code characters.

Note 55-K05: Fixed problem where it was not possible to shut down Windows while an application was running

Fixed a problem where it was not possible to shut down Windows or log off when a NITRO-SoundMaker application was running.

Note 55-K06: Fixed a problem where an invalid instrument is generated

Fixed a problem where an instrument with the program number -1 was generated when creating a new instrument using the context menu in the instrument list margin. Corrected so that a new instrument cannot be created in the instrument list margin.
