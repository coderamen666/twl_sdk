Changes in 2.2.0 PR

Note 83-X01: Ended support for 3ds Max 8.0

Ended support for 3ds Max 8.0.

Changes in 2.1.1

Note 82-X01: 3ds Max version

This update does not contain the plug-in for 3ds Max 8.0. Also, support is scheduled to be provided for 3ds Max. 9.0 and later at the next update.

Note 82-X02: Support for 3ds Max 2010 (Windows 32-bit version)

There is now support for 3ds Max 2010 (Windows 32-bit version).

Note 82-X03: Corrected problems in NITRO Replace Character and NITRO Rename Over 16 Characters

Corrected a problem where it was not possible to change the material name applied to nodes that have parent nodes, using NITRO Replace Character and NITRO Rename Over 16 Characters.

Changes in 2.1.0

Note 81-X01: (Manual) Revised Material Types and Attributes

Revised the cautionary note regarding opacity in Material Types and Attributes.

Note 81-X02: Revised so that an error occurs when a node name or material name is blank

Revised so that now an error occurs when a node name or a material name is left blank.

Note 81-X03: When mesh does not have polygons, warning is now displayed, and NULL is output

When a mesh does not have polygons, a warning is now displayed, and NULL is output.

Note 81-X04: Fixed invalid modifier bug

When an error occurred during output, sometimes the modifier would become invalid. That bug has been fixed.

Note 81-X05: Fixed bug in Physique export

When Physique was used for a model, sometimes the pose in the IMD file would not be output correctly. That bug has been fixed.

Changes in 2.0.0

Note 80-X01: Accelerated exporting with specified frame range

Exporting with a specified frame range has been accelerated.

Note 80-X02: Fixed bug when Merge Useless Node specified

When Merge Useless Node was specified, sometimes the <node> child would not be correctly exported. That bug was fixed.

Note 80-X03: Fixed bug in Physique export

When Physique was used for a model, the pose of the IMD file would not be exported correctly. That bug was fixed.
